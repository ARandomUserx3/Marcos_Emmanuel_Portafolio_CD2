01 · ¿Qué es CSS?

CSS (Cascading Style Sheets — Hojas de Estilo en Cascada) es el lenguaje que controla la presentación visual de las páginas web. Mientras HTML define la estructura, CSS define cómo se ve: colores, tamaños, fuentes, espaciado, animaciones y más.

🌊 ¿Qué significa "cascada"?

La "C" de CSS hace referencia a que los estilos se aplican en cascada: si hay reglas que entran en conflicto, el navegador sigue un orden de especificidad y prioridad para decidir cuál aplica.

3️⃣ ¿Qué añade CSS3?

CSS3 es la versión actual. Introdujo módulos independientes: bordes redondeados, sombras, gradientes, transiciones, animaciones, Flexbox, Grid, variables y mucho más, sin necesitar JavaScript.

💡 Regla CSS básica

Una regla CSS tiene dos partes: el selector (a qué elemento apunta) y la declaración (qué estilo aplica). La declaración es un par propiedad: valor; encerrado en llaves.

02 · Formas de Usar CSS

formas-de-css.html
/* ── 1. CSS EXTERNO (recomendado) ─────────────────────── */
/* En el <head> del HTML se enlaza el archivo .css */
<link rel="stylesheet" href="estilos.css"/>

/* ── 2. CSS INTERNO (en el mismo HTML) ────────────────── */
<style>
  body {
    background: #0b0f1a;
    color: white;
  }
</style>

/* ── 3. CSS EN LÍNEA (inline — evitar) ────────────────── */
<p style="color: red; font-size: 18px;">Texto rojo</p>
CSS Externo ✅
Un solo archivo .css para todo el sitio. Fácil de mantener y reutilizar. El método recomendado siempre.
CSS Interno ⚠️
Útil para estilos de una sola página o prototipos rápidos. No ideal para sitios completos.
CSS Inline ❌
Difícil de mantener, no reutilizable, mezcla estructura con estilo. Úsalo solo en casos muy específicos.

03 · Selectores

Los selectores son los patrones que usa CSS para encontrar y apuntar a los elementos HTML que quieres estilizar. Es la parte más importante de aprender CSS.

selectores.css
/* ── BÁSICOS ─────────────────────────────────────────── */
*           { margin: 0; }    /* Universal: todos los elementos */
p           { color: white; }  /* Etiqueta: todos los <p> */
.card       { padding: 20px; } /* Clase: elementos con class="card" */
#menu       { display: flex; } /* ID: el elemento con id="menu" */

/* ── COMBINADORES ───────────────────────────────────── */
nav a       { color: blue; }   /* Descendiente: <a> dentro de <nav> */
ul > li     { color: red; }    /* Hijo directo: <li> hijo inmediato de <ul> */
h1 + p      { margin: 0; }     /* Hermano adyacente: <p> justo después de <h1> */
h1 ~ p      { opacity: .8; }  /* Hermanos generales: todos los <p> tras <h1> */

/* ── ATRIBUTOS ──────────────────────────────────────── */
input[type="text"]    { border: 1px solid blue; }
a[href^="https"]   { color: green; }  /* empieza con */
a[href$=".pdf"]    { color: red; }    /* termina en */
img[alt*="logo"]    { width: 80px; }   /* contiene */

/* ── MÚLTIPLES ──────────────────────────────────────── */
h1, h2, h3   { font-weight: bold; }  /* Aplica a los tres */

La especificidad determina qué regla gana cuando varias apuntan al mismo elemento. Se calcula con tres categorías:

1-0-0
ID
#id
›
0-1-0
Clase / Atributo
.class [attr]
›
0-0-1
Etiqueta
p, div, h1
›
!
!important
evitar siempre

04 · El Modelo de Caja (Box Model)

Todo elemento HTML es una caja rectangular. El Box Model describe cómo se compone esa caja de cuatro capas concéntricas.

margin
border
padding
content width × height
box-model.css
.caja {
  /* Tamaño del contenido */
  width:  300px;
  height: 200px;

  /* Relleno interior (empuja el contenido hacia adentro) */
  padding:        20px;           /* todos los lados */
  padding:        10px 20px;      /* vertical | horizontal */
  padding:        5px 10px 15px 20px; /* top right bottom left */
  padding-top:    10px;

  /* Borde */
  border:         2px solid #264de4;
  border-radius:  12px;
  border-top:     4px dashed red;

  /* Margen exterior (espacio entre elementos) */
  margin:         20px;
  margin:         0 auto;          /* centrar horizontalmente */

  /* box-sizing: incluye padding y border en el width */
  box-sizing:     border-box;     /* ← usar siempre */
}
⚠️ Truco esencial

Agrega *, *::before, *::after { box-sizing: border-box; } al inicio de todo proyecto CSS. Esto hace que el width que defines incluya padding y border, evitando sorpresas de tamaño.

05 · Colores y Fondos

colores-fondos.css
.elemento {
  /* Formatos de color */
  color: red;                     /* nombre */
  color: #264de4;                 /* hexadecimal */
  color: rgb(38, 77, 228);          /* rgb */
  color: rgba(38, 77, 228, 0.5);    /* rgba (con opacidad) */
  color: hsl(228, 75%, 52%);        /* hsl (tono, saturación, luminosidad) */
  color: hsla(228, 75%, 52%, .8);  /* hsla */
  opacity: 0.8;                   /* transparencia del elemento */

  /* Fondos */
  background-color: #111827;
  background-image: url('imagen.jpg');
  background-size:   cover;        /* cover | contain | 100px */
  background-position: center;
  background-repeat:   no-repeat;

  /* Gradientes CSS3 */
  background: linear-gradient(135deg, #264de4, #00c4ff);
  background: radial-gradient(circle, #264de4, transparent);

  /* Sombras (CSS3) */
  box-shadow:  0 4px 20px rgba(0,0,0,.4);
  text-shadow: 2px 2px 8px black;
}

06 · Tipografía

tipografia.css
/* Importar fuente de Google Fonts */
@import url('https://fonts.googleapis.com/css2?family=Montserrat:wght@400;700');

/* Cargar fuente local con @font-face */
@font-face {
  font-family: 'MiFuente';
  src: url('fuente.woff2') format('woff2');
}

p {
  font-family:    'Montserrat', sans-serif;
  font-size:      16px;           /* px | em | rem | % | vw */
  font-weight:    400;            /* 100–900 o normal/bold */
  font-style:     italic;         /* normal | italic | oblique */
  line-height:    1.6;            /* alto de línea, sin unidad es relativo */
  letter-spacing: 0.05em;        /* espacio entre letras */
  text-align:     center;         /* left | right | center | justify */
  text-transform: uppercase;      /* uppercase | lowercase | capitalize */
  text-decoration: underline;     /* none | underline | line-through */
  white-space:    nowrap;         /* no permitir saltos de línea */
}

/* Unidades de medida importantes */
/* px  → píxeles fijos                        */
/* em  → relativo al font-size del padre       */
/* rem → relativo al font-size del <html>      */
/* %   → relativo al elemento padre            */
/* vw/vh → % del ancho/alto de la pantalla     */
/* clamp(min, ideal, max) → responsive fluido  */
font-size: clamp(1rem, 4vw, 3rem);

display.css
.elemento {

  /* Por defecto, flujo normal   */

08 · Flexbox

Flexbox es el sistema de layout unidimensional de CSS. Permite distribuir y alinear elementos en una fila o columna de forma flexible y eficiente.

item 1
item 2
item 3

↑ justify-content: space-between

flexbox.css
/* ── CONTENEDOR FLEX ─────────────────────────────────── */
.contenedor {
  display:         flex;

  /* Dirección del eje principal */
  flex-direction:  row;          /* row | row-reverse | column | column-reverse */

  /* Alineación en el eje principal (horizontal si row) */
  justify-content: center;      /* flex-start | flex-end | center | space-between | space-around | space-evenly */

  /* Alineación en el eje cruzado (vertical si row) */
  align-items:     center;      /* flex-start | flex-end | center | stretch | baseline */

  /* Permitir salto de línea */
  flex-wrap:       wrap;         /* nowrap | wrap | wrap-reverse */

  /* Espacio entre items */
  gap:             16px;
}

/* ── ITEMS FLEX ──────────────────────────────────────── */
.item {
  /* Cuánto puede crecer */
  flex-grow:   1;     /* 0 = no crece, 1 = crece proporcionalmente */

  /* Cuánto puede encogerse */
  flex-shrink: 1;

  /* Tamaño base antes de distribuir espacio */
  flex-basis:  200px;

  /* Abreviación: grow shrink basis */
  flex:        1 1 auto;

  /* Alineación individual (sobreescribe align-items) */
  align-self:  flex-end;
}

/* ── PATRÓN COMÚN: centrar todo ─────────────────────── */
.centrar {
  display:         flex;
  justify-content: center;
  align-items:     center;
}

09 · CSS Grid

CSS Grid es el sistema bidimensional de CSS: trabaja con filas y columnas al mismo tiempo. Ideal para layouts completos de página.

header (col span 2)
nav
main
aside
footer (col span 3)
grid.css
/* ── CONTENEDOR GRID ─────────────────────────────────── */
.layout {
  display:               grid;

  /* Definir columnas */
  grid-template-columns:  200px 1fr 1fr;     /* 3 columnas */
  grid-template-columns:  repeat(3, 1fr);      /* 3 iguales */
  grid-template-columns:  repeat(auto-fill, minmax(200px, 1fr)); /* responsive */

  /* Definir filas */
  grid-template-rows:     auto 1fr auto;

  /* Espaciado */
  gap:                    20px;
  column-gap:             24px;
  row-gap:                16px;
}

/* ── POSICIONAR ITEMS ────────────────────────────────── */
.header { grid-column: 1 / -1; }         /* desde col 1 hasta la última */
.sidebar{ grid-column: 1; grid-row: 2 / 4; } /* ocupa 2 filas */
.card   { grid-column: span 2; }          /* abreviado */

/* ── ALINEACIÓN EN GRID ──────────────────────────────── */
.layout {
  justify-items:   center;  /* alinea contenido horizontal en celda */
  align-items:     center;  /* alinea contenido vertical en celda */
  place-items:     center;  /* shorthand para ambos */
}

11 · Transiciones y Animaciones

transiciones.css
/* Transición: cambio suave entre estados */
.boton {
  background:  #264de4;
  transition:  all .3s ease;

  /* propiedad | duración | timing */
  transition:
    background .2s ease,
    transform  .3s ease;
}

.boton:hover {
  background:  #00c4ff;
  transform:   translateY(-4px)
               scale(1.05);
}

/* Funciones de timing */
/* ease | linear | ease-in      */
/* ease-out | ease-in-out       */
/* cubic-bezier(0,.68,.27,1.55) */
animaciones.css
/* @keyframes: define la animación */
@keyframes aparecer {
  0%   { opacity: 0; transform: translateY(20px); }
  100% { opacity: 1; transform: translateY(0); }
}

@keyframes girar {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

/* Aplicar la animación */
.tarjeta {
  animation-name:     aparecer;
  animation-duration: .6s;
  animation-delay:    .2s;
  animation-iteration-count: 1;
  animation-fill-mode: both;

  /* Abreviado */
  animation: aparecer .6s ease .2s 1 both;
}

12 · Variables CSS (Custom Properties)

Las variables CSS (introducidas en CSS3) permiten guardar valores reutilizables. Son fundamentales para crear temas consistentes y fáciles de mantener.

variables.css
/* Declarar variables en :root (disponibles en todo el doc) */
:root {
  /* Colores */
  --color-primario:   #264de4;
  --color-fondo:      #0b0f1a;
  --color-texto:      #f0f4ff;
  --color-muted:      rgba(255,255,255,.5);

  /* Tamaños */
  --radio-borde:      12px;
  --espaciado:        20px;
  --fuente-base:      1rem;
}

/* Usar variables con var() */
.tarjeta {
  background:    var(--color-fondo);
  color:         var(--color-texto);
  border-radius: var(--radio-borde);
  padding:       var(--espaciado);

  /* Con valor fallback por si la variable no existe */
  border: 1px solid var(--borde, rgba(255,255,255,.1));
}

/* Tema oscuro / claro con variables */
[data-tema="claro"] {
  --color-fondo: #ffffff;
  --color-texto: #111827;
}

13 · Media Queries (Diseño Responsivo)

Las media queries permiten aplicar estilos según el tamaño de pantalla. Son la base del diseño responsivo: que la página se vea bien en celular, tablet y computadora.

responsive.css
/* ── BREAKPOINTS COMUNES ─────────────────────────────── */
/*  < 480px  → Móvil pequeño                             */
/*  < 768px  → Móvil / Tablet vertical                   */
/* < 1024px  → Tablet horizontal / Laptop pequeño        */
/* < 1280px  → Desktop                                   */

/* Estrategia Mobile First: estilos base = móvil */
.grid {
  display: grid;
  grid-template-columns: 1fr;  /* 1 columna en móvil */
  gap: 16px;
}

/* Tablet: 2 columnas */
@media (min-width: 768px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}

/* Desktop: 3 columnas */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* Otros tipos de media query */
@media (orientation: landscape) { /* pantalla horizontal */ }
@media (prefers-color-scheme: dark) { /* modo oscuro del SO */ }
@media (prefers-reduced-motion: reduce) {
  /* desactivar animaciones si el usuario lo prefiere */
  * { animation: none !important; }
}
@media (max-width: 600px) {
  .sidebar { display: none; }  /* ocultar barra lateral en móvil */
}

14 · Buenas Prácticas

1

Usa box-sizing: border-box globalmente

Añade *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; } al inicio. Esto evita inconsistencias de tamaño y resetea márgenes por defecto del navegador.

2

Variables CSS para colores y tamaños

Define todos tus colores, fuentes y espaciados en :root con variables. Cambiar el tema completo de un sitio se vuelve trivial: solo editas las variables.

3

Diseña Mobile First

Escribe los estilos base para pantallas pequeñas y usa min-width en media queries para añadir complejidad en pantallas más grandes. Es más fácil añadir que quitar.

4

Evita !important

El uso de !important rompe la cascada y hace el CSS impredecible y difícil de depurar. Si lo necesitas, es señal de que tus selectores están mal organizados.

5

Usa clases, no IDs para estilos

Los IDs tienen especificidad muy alta y solo pueden usarse una vez por página. Las clases son reutilizables y más fáciles de sobrescribir cuando es necesario.

6

Organiza tu CSS en secciones

Divide tu archivo CSS con comentarios: Reset, Variables, Tipografía, Layout, Componentes, Utilidades, Media Queries. Esto facilita enormemente el mantenimiento.
