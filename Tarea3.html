01 · ¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje de marcado estándar para crear páginas web. No es un lenguaje de programación: no tiene lógica ni variables. Su función es describir la estructura y el contenido de una página usando etiquetas.

🌐 ¿Para qué sirve?

HTML define qué hay en una página: títulos, párrafos, imágenes, botones, formularios y más. El navegador lee el HTML y lo convierte en lo que ves en pantalla.

📅 ¿Qué es HTML5?

HTML5 es la versión actual del estándar. Introducida en 2014, añade nuevas etiquetas semánticas, soporte nativo para audio y video, canvas, y APIs de geolocalización y almacenamiento.

💡 Dato clave

Una página web está construida con tres tecnologías: HTML (estructura), CSS (estilo) y JavaScript (comportamiento). HTML es siempre el primero que debes aprender.

02 · Estructura Base de un Documento

Todo documento HTML5 válido debe contener esta estructura mínima. Cada parte tiene un rol específico:

mi-primera-pagina.html 
<!-- 1. Declara que es HTML5 -->
<!DOCTYPE html>

<!-- 2. Raíz del documento, define el idioma -->
<html lang="es">

  <!-- 3. Metadatos (no visibles en pantalla) -->
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <title>Título de mi página</title>
    <link rel="stylesheet" href="estilos.css"/>
  </head>

  <!-- 4. Contenido visible -->
  <body>
    <h1>¡Hola Mundo!</h1>
    <p>Mi primera página web.</p>
  </body>

</html>
<!DOCTYPE html>
Le dice al navegador que este es un documento HTML5. Va siempre al inicio.
<html lang="es">
Elemento raíz. El atributo lang define el idioma del contenido.
<head>
Contiene metadatos: título, codificación, CSS vinculado, descripciones SEO.
<meta charset>
Define la codificación de caracteres. UTF-8 permite tildes, ñ y símbolos.
<meta viewport>
Hace la página responsive en celulares. Obligatorio en HTML5 moderno.
<body>
Todo el contenido visible de la página va dentro de este elemento.

03 · Etiquetas de Texto

Las etiquetas de texto son las más usadas en HTML. Estructuran y dan formato al contenido escrito de la página.

textos.html 
<!-- Encabezados (h1 es el más importante, h6 el menor) -->
<h1>Título Principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>
<h4>Subsección</h4>
<h5>Nivel 5</h5>
<h6>Nivel 6</h6>

<!-- Párrafo -->
<p>Esto es un párrafo de texto normal.</p>

<!-- Énfasis y negritas -->
<p>Texto <strong>importante (negrita)</strong> y texto <em>enfatizado (cursiva)</em>.</p>

<!-- Otros elementos de texto -->
<p>Texto <mark>resaltado</mark>, <del>tachado</del>, <ins>insertado</ins>.</p>
<p>H<sub>2</sub>O y E=mc<sup>2</sup></p>

<!-- Código y preformateado -->
<code>console.log('Hola')</code>
<pre>  Texto con
  espacios exactos</pre>

<!-- Salto de línea y línea horizontal -->
<br/>  <!-- salto de línea -->
<hr/>  <!-- línea separadora -->

<!-- Cita -->
<blockquote>La web para todos. — Tim Berners-Lee</blockquote>
<h1>–<h6>
Encabezados jerárquicos. Solo debe haber un <h1> por página (SEO).
<p>
Párrafo. El elemento de texto más común. Agrega espacio arriba y abajo.
<strong>
Texto importante, se muestra en negrita. Tiene significado semántico.
<em>
Énfasis, se muestra en cursiva. Diferente de <i> que es solo visual.
<mark>
Resalta texto como con marcador amarillo. Nuevo en HTML5.
<code> / <pre>
Muestra código fuente en fuente monoespaciada, respetando espacios.

05 · Enlaces e Imágenes

enlaces-imagenes.html 
<!-- Enlace básico -->
<a href="https://www.google.com">Ir a Google</a>

<!-- Abrir en nueva pestaña -->
<a href="https://google.com" target="_blank" rel="noopener">Nueva pestaña</a>

<!-- Enlace a sección de la misma página (ancla) -->
<a href="#seccion2">Ir a sección 2</a>
<div id="seccion2">...</div>

<!-- Enlace de email y teléfono -->
<a href="mailto:correo@ejemplo.com">Enviar correo</a>
<a href="tel:+521234567890">Llamar</a>

<!-- Imagen -->
<img
  src="foto.jpg"
  alt="Descripción de la imagen"
  width="400"
  height="300"
  loading="lazy"
/>

<!-- Imagen como enlace -->
<a href="pagina.html">
  <img src="logo.png" alt="Logo"/>
</a>
⚠️ Importante

El atributo alt en las imágenes es obligatorio para accesibilidad y SEO. Describe lo que muestra la imagen para personas con lectores de pantalla. Si la imagen es decorativa, usa alt="".

06 · Tablas

Las tablas se usan para mostrar datos tabulares (como una hoja de cálculo). No deben usarse para el diseño o layout de la página.

tabla.html 
<table>
  <!-- Cabecera de la tabla -->
  <thead>
    <tr>
      <th>Lenguaje</th>
      <th>Tipo</th>
      <th>Año</th>
    </tr>
  </thead>

  <!-- Cuerpo de la tabla -->
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Marcado</td>
      <td>1993</td>
    </tr>
    <tr>
      <td>CSS</td>
      <td>Estilos</td>
      <td>1996</td>
    </tr>
    <!-- colspan une columnas, rowspan une filas -->
    <tr>
      <td colspan="2">JavaScript</td>
      <td>1995</td>
    </tr>
  </tbody>

  <!-- Pie de tabla -->
  <tfoot>
    <tr>
      <td colspan="3">Fuente: W3C</td>
    </tr>
  </tfoot>
</table>

07 · Formularios

Los formularios permiten al usuario ingresar datos. HTML5 añadió muchos tipos de input nuevos con validación nativa.

formulario.html 
<form action="/enviar" method="post">

  <!-- Texto simple -->
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre" placeholder="Tu nombre" required/>

  <!-- Tipos HTML5 con validación automática -->
  <input type="email"    placeholder="correo@ejemplo.com"/>
  <input type="password" placeholder="Contraseña"/>
  <input type="number"   min="1" max="100"/>
  <input type="date"/>
  <input type="color"/>
  <input type="range"    min="0" max="10"/>
  <input type="file"     accept=".pdf,.png"/>

  <!-- Checkbox y radio -->
  <input type="checkbox" id="acepto"/> <label for="acepto">Acepto términos</label>
  <input type="radio" name="genero" value="m"/> Masculino
  <input type="radio" name="genero" value="f"/> Femenino

  <!-- Selector desplegable -->
  <select name="ciudad">
    <option value="tuxtla">Tuxtla Gutiérrez</option>
    <option value="cdmx">Ciudad de México</option>
  </select>

  <!-- Área de texto -->
  <textarea rows="4" placeholder="Tu mensaje..."></textarea>

  <!-- Botones -->
  <button type="submit">Enviar</button>
  <button type="reset">Limpiar</button>

</form>

08 · HTML Semántico

HTML5 introdujo etiquetas semánticas que describen el significado del contenido, no solo su apariencia. Mejoran el SEO, accesibilidad y mantenimiento del código.

semantica.html 
<body>

  <!-- Encabezado principal del sitio -->
  <header>
    <nav>
      <a href="/">Inicio</a>
      <a href="/acerca">Acerca</a>
    </nav>
  </header>

  <!-- Contenido principal (solo uno por página) -->
  <main>

    <!-- Artículo independiente (blog, noticia) -->
    <article>
      <h1>Título del artículo</h1>
      <p>Contenido del artículo...</p>
    </article>

    <!-- Sección temática -->
    <section>
      <h2>Últimas noticias</h2>
    </section>

    <!-- Contenido relacionado / barra lateral -->
    <aside>
      <p>Artículos relacionados...</p>
    </aside>

  </main>

  <!-- Pie de página -->
  <footer>
    <p>&copy; 2026 Mi Sitio</p>
  </footer>

</body>
<header>
Encabezado de la página o de una sección. Suele contener logo y navegación.
<nav>
Bloque de navegación principal. Para menús y enlaces de navegación.
<main>
Contenido principal único de la página. Solo debe existir uno por documento.
<article>
Contenido independiente: post, noticia, comentario. Tiene sentido solo.
<section>
Agrupa contenido relacionado bajo un mismo tema. Siempre con encabezado.
<aside>
Contenido lateral relacionado pero no esencial: widgets, publicidad, notas.
<footer>
Pie de la página o sección. Créditos, enlaces legales, contacto.
<figure>
Agrupa una imagen con su descripción <figcaption>.
<time>
Representa una fecha u hora legible para máquinas con atributo datetime.

10 · Atributos Globales

Los atributos globales pueden usarse en cualquier etiqueta HTML. Son fundamentales para CSS, JavaScript y accesibilidad.

idid="menu"Identificador único. Para CSS y JS. No repetir en la página.
classclass="card activo"Clase(s) para CSS y JS. Puede repetirse y tener múltiples valores.
stylestyle="color:red"Estilos en línea. Úsalo poco; mejor usar CSS externo.
titletitle="Más info"Tooltip que aparece al pasar el cursor sobre el elemento.
hiddenhiddenOculta el elemento. El navegador no lo muestra ni lo procesa.
tabindextabindex="0"Controla el orden de tabulación para accesibilidad con teclado.
langlang="en"Define el idioma de ese elemento específico dentro de la página.
data-*data-id="42"Atributos personalizados para pasar datos a JavaScript.
contenteditablecontenteditable="true"Permite que el usuario edite el texto del elemento directamente.
draggabledraggable="true"Hace el elemento arrastrable con drag & drop.

11 · Características Nuevas de HTML5

HTML5 no solo añadió etiquetas semánticas. Trajo nuevas APIs y elementos que expanden lo que puedes hacer solo con el navegador.

canvas.html 
<!-- Canvas: dibujo 2D con JS -->
<canvas
  id="lienzo"
  width="400"
  height="200"
></canvas>

<script>
const c = document
  .getElementById('lienzo')
  .getContext('2d');
c.fillStyle = '#e34c26';
c.fillRect(10,10,100,80);
</script>
details-dialog.html 
<!-- Details / Summary -->
<details>
  <summary>Ver más</summary>
  <p>Contenido oculto...</p>
</details>

<!-- Dialog nativo -->
<dialog id="modal">
  <p>¡Soy un modal!</p>
  <button
    onclick="modal.close()"
  >Cerrar</button>
</dialog>

🗺 Geolocalización

La API navigator.geolocation permite obtener la posición GPS del usuario con su permiso.

💾 localStorage

Almacena datos en el navegador sin necesidad de servidor. Persiste aunque se cierre la pestaña.

🔌 Web Workers

Ejecuta scripts en segundo plano sin bloquear la interfaz. Para tareas pesadas de procesamiento.

12 · Buenas Prácticas

Seguir estas reglas desde el inicio hará que tu código sea profesional, accesible y fácil de mantener.

1

Siempre usa DOCTYPE y lang

Escribe <!DOCTYPE html> en la primera línea y define el idioma con lang="es" en la etiqueta html. Esto es obligatorio en HTML5 válido.

2

Usa etiquetas semánticas, no solo divs

Evita llenar tu página de <div>. Usa <header>, <main>, <section>, <article>, <footer>. Los motores de búsqueda y lectores de pantalla lo agradecen.

3

Cierra todas las etiquetas

Aunque el navegador a veces perdona etiquetas sin cerrar, es mala práctica. Cierra todo explícitamente: <p>texto</p>. Las etiquetas vacías llevan />: <br/>, <img/>.

4

Usa minúsculas y comillas en atributos

Escribe etiquetas y atributos en minúsculas: <div class="caja">. Siempre pon los valores de atributos entre comillas dobles.

5

No uses atributos visuales en HTML

Evita align, bgcolor, border, font. Esos eran de HTML4. En HTML5 todo el estilo visual va en CSS.

6

Valida tu HTML

Usa el validador oficial del W3C en validator.w3.org para verificar que tu código no tenga errores. Un HTML válido funciona mejor en todos los navegadores.
