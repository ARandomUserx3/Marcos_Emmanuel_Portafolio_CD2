01 · ¿Qué es Python?

Python es un lenguaje de programación de propósito general, de alto nivel e interpretado, creado por Guido van Rossum en 1991. Su filosofía central es la legibilidad del código: que leer Python sea casi como leer inglés. Hoy es el lenguaje más popular del mundo, dominando en IA, ciencia de datos, automatización y desarrollo web.

✅ ¿Por qué aprender Python?

Sintaxis limpia y simple, ideal para principiantes. Enormes comunidad y ecosistema de librerías. Es el lenguaje estándar para Machine Learning e Inteligencia Artificial. Versátil: sirve para web, datos, automatización, scripts y más.

⚡ Características clave

Interpretado: se ejecuta línea a línea sin compilar. Tipado dinámico: no declaras el tipo de variable. Multiparadigma: soporta programación orientada a objetos, funcional e imperativa. Multiplataforma: corre en Windows, Mac y Linux.

Usos principales de Python en la actualidad:

📊 Ciencia de datos

⚙️ Automatización y scripts
🔬 Investigación científica

🔒 Ciberseguridad
☁️ Cloud computing

02 · Instalación y Entorno

1. Instalar Python
Descarga la versión más reciente desde python.org. En Windows activa "Add Python to PATH" durante la instalación.
2. Editor de código
VS Code + extensión Python es la opción más recomendada. También puedes usar PyCharm, Thonny (ideal para principiantes) o IDLE.
3. Entornos virtuales
Usa python -m venv env para crear un entorno aislado por proyecto y evitar conflictos de librerías.

03 · Sintaxis Básica

La característica más importante de Python es que usa indentación (sangría) para definir bloques de código, en lugar de llaves {} como en otros lenguajes.

sintaxis_basica.py
# Esto es un comentario de una línea

"""
Este es un comentario
de múltiples líneas (docstring)
"""

# Imprimir en pantalla
print("¡Hola, Mundo!")
print("Hola", "Marcos", sep=", ")

# Leer entrada del usuario
nombre = input("¿Cuál es tu nombre? ")
print(f"Hola, {nombre}!")   # f-string: cadena con variables

# La indentación define bloques — ¡muy importante!
if nombre == "Marcos":
    print("¡Bienvenido, Marcos!")   # 4 espacios = dentro del bloque
    print("Este bloque está indentado")
print("Esto está fuera del if")       # sin indentación = fuera
⚠️ Regla de oro

En Python la indentación no es opcional: es parte de la sintaxis. Usa siempre 4 espacios por nivel (no tabulaciones). Mezclar espacios y tabs causa errores.

06 · Estructuras de Datos

estructuras.py
# ── LISTA (list) — ordenada, mutable ─────────────────────
frutas = ["manzana", "pera", "uva"]
frutas.append("mango")      # añadir al final
frutas.insert(1, "kiwi")    # insertar en posición
frutas.remove("pera")       # eliminar por valor
frutas.pop()                # eliminar el último
frutas.sort()               # ordenar
print(frutas[0])            # primer elemento
print(frutas[-1])           # último elemento
print(frutas[1:3])          # slicing

# List comprehension — forma pythónica de crear listas
cuadrados = [x**2 for x in range(10)]
pares     = [x for x in range(20) if x % 2 == 0]

# ── TUPLA (tuple) — ordenada, inmutable ──────────────────
coordenadas = (16.75, -93.11)    # no se puede modificar
lat, lng    = coordenadas           # desempaquetado

# ── DICCIONARIO (dict) — clave:valor, mutable ────────────
alumno = {
    "nombre": "Marcos",
    "grado":  3,
    "activo": True
}
print(alumno["nombre"])         # Marcos
print(alumno.get("edad", 0))    # 0 si no existe
alumno["edad"] = 17             # añadir/actualizar
del alumno["activo"]           # eliminar clave
print(alumno.keys())           # claves
print(alumno.values())         # valores
print(alumno.items())          # pares (clave, valor)

# ── SET (conjunto) — sin duplicados, no ordenado ─────────
colores = {"rojo", "azul", "rojo"}   # {"rojo", "azul"}
colores.add("verde")
print("rojo" in colores)         # True (muy rápido)
list []
Ordenada, mutable, permite duplicados. La más usada. Ideal para colecciones que cambian.
tuple ()
Ordenada, inmutable. Más rápida que list. Ideal para datos que no deben cambiar.
dict {}
Pares clave-valor. Acceso instantáneo por clave. La estructura más útil en Python.
set {}
Sin duplicados, no ordenado. Ideal para verificar pertenencia y operaciones de conjuntos.
range()
Secuencia de números. Muy eficiente en memoria. Base de los bucles for.
str
Secuencia inmutable de caracteres. Soporta indexación, slicing y decenas de métodos.

07 · Control de Flujo

condicionales.py
# if / elif / else
nota = 85

if nota >= 90:
    print("Excelente")
elif nota >= 70:
    print("Aprobado")
elif nota >= 60:
    print("Suficiente")
else:
    print("Reprobado")

# match/case (Python 3.10+)
match nota:
    case n if n >= 90: print("A")
    case n if n >= 70: print("B")
    case _:            print("F")
bucles.py
# for — iterar sobre secuencias
for i in range(5):             # 0,1,2,3,4
    print(i)

for fruta in ["🍎","🍊","🍇"]:
    print(fruta)

# enumerate — índice + valor
for i, v in enumerate(["a","b"]):
    print(i, v)

# while — mientras condición sea True
contador = 0
while contador < 3:
    print(contador)
    contador += 1

# break / continue / else
for n in range(10):
    if n == 3: continue   # saltar
    if n == 6: break      # salir
    print(n)

def potencia(base, exponente=2):   # valor por defecto
    return base ** exponente

    return sum(numeros)

    for clave, valor in datos.items():

perfil(nombre="Marcos", grado=3)

# ── FUNCIONES LAMBDA (anónimas) ──────────────────────────
cuadrado = lambda x: x ** 2
print(cuadrado(5))   # 25

    return min(lista), max(lista)   # retorna tupla

10 · Programación Orientada a Objetos

La POO organiza el código en clases (plantillas) y objetos (instancias). Permite modelar conceptos del mundo real con sus atributos (datos) y métodos (comportamientos).

poo.py
# ── CLASE Y OBJETO ───────────────────────────────────────
class Alumno:
    """Clase que representa a un alumno."""

    escuela = "CBTIS 144"     # atributo de clase (compartido)

    def __init__(self, nombre, grado, promedio):
        # self = referencia al propio objeto
        self.nombre   = nombre     # atributo de instancia
        self.grado    = grado
        self.promedio = promedio

    def presentarse(self):       # método de instancia
        return f"Soy {self.nombre}, grado {self.grado}"

    def aprobo(self):
        return self.promedio >= 60

    def __str__(self):           # método especial (dunder)
        return f"Alumno({self.nombre}, {self.grado})"

# Crear objetos (instancias)
marcos = Alumno("Marcos", 3, 88)
print(marcos.presentarse())
print(marcos.aprobo())         # True
print(Alumno.escuela)          # CBTIS 144

# ── HERENCIA ─────────────────────────────────────────────
class Becario(Alumno):           # hereda de Alumno
    def __init__(self, nombre, grado, promedio, beca):
        super().__init__(nombre, grado, promedio)
        self.beca = beca

    def presentarse(self):       # sobreescribir método
        base = super().presentarse()
        return f"{base} — Becario: {self.beca}"

becario = Becario("Luis", 2, 95, "Excelencia")
print(becario.presentarse())

11 · Manejo de Errores

errores.py
# try / except / else / finally
try:
    numero = int(input("Número: "))
    resultado = 10 / numero

except ValueError:
    print("No es un número válido")

except ZeroDivisionError:
    print("No se puede dividir entre 0")

except Exception as e:
    print(f"Error inesperado: {e}")

else:
    print(f"Resultado: {resultado}")

finally:
    print("Esto siempre se ejecuta")

13 · Buenas Prácticas

1

Sigue PEP 8 — la guía de estilo oficial

PEP 8 define las convenciones de Python: usa 4 espacios por indentación, nombres en snake_case, constantes en MAYÚSCULAS, clases en PascalCase y líneas de máximo 79 caracteres.

2

Escribe docstrings en tus funciones y clases

Documenta siempre tus funciones con """triple comillas""". Explica qué hace, qué recibe y qué devuelve. Es la base de código mantenible y profesional.

3

Usa f-strings en lugar de concatenación

En vez de "Hola " + nombre, usa f"Hola {nombre}". Son más legibles, rápidas y permiten expresiones dentro: f"Área: {pi * r**2:.2f}".

4

Usa list comprehensions cuando sea claro

[x**2 for x in range(10)] es más pythónico que un loop de 4 líneas. Pero si la expresión se vuelve difícil de leer, usa el loop tradicional.

5

Maneja siempre las excepciones específicas

Atrapa excepciones específicas (except ValueError) en lugar de usar except: a secas. Capturar todo oculta errores reales y hace la depuración muy difícil.

6

Practica todos los días con proyectos reales

La única forma de aprender Python es programando. Empieza con scripts pequeños: calculadora, juego de adivinanza, lista de tareas. Luego avanza a análisis de datos y proyectos de IA.
